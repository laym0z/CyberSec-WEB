Injection

SQL ін'єкція - це вразливість, яка дозволяє звичайному користувачеві маніпулювати базою даних сервера, тим самим: отримувати важливу приховану інформацію, яка доступна не всім, видаляти і видозмінювати. Саме тому критично важливо завжди перевіряти всі дані введені користувачами. 

SELECT * FROM users WHERE username = "user_input_username" and password = "user_input_password"

В наведені вище поля користувач міг би вписати дані, які могли б стати частиною SQL запиту. І тоді, користувач міг би, на приклад, обійти систему авторизації не знаючи пароль до користувача.

Така вразливість можлива, якщо виконувати SQL запити у сирому вигляді. В моєму застосунку використовується SQLAlchemy, і в ньому є можливість використовувати конструктор запитів.

Завдяки SQLAlchemy бібліотеці я можу створювати моделі моїх таблиць, які зберігатимуться в PostgreSQL, та дуже зручно працювати з методами цієї бібліотеки для того, аби маніпулювати даними:

class Users(UserMixin, db.Model):
    __tablename__ = "users"

    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(80), unique=True, nullable=False)
    email = db.Column(db.String(80), unique=True, nullable=False)
    password_hash = db.Column(db.String(128), nullable=False)
    avatar = db.Column(db.String(256))
    created_at = db.Column(db.DateTime, default=datetime.now())

user = Users.query.filter_by(username=sanitized_login).first()

Якщо відображати рядок коду вище в сирому SQL запиті, то він буде мати наступний вигляд:

SELECT * FROM users WHERE username=:sanitized_login;

Ну а на рівні БД так, а введені дані передаватимуться окремо:

SELECT * FROM users WHERE username= ?

Тобто параметри обробляються окремо і не є частиною SQL коду

Але якщо вже ну дуже треба написати SQL запит вручну, то варто робити так:

stmt = text("SELECT * FROM users WHERE username = :username")
db.session.execute(stmt, {"username": login})

Ми передаємо дані саме як ПАРАМЕТР, не треба робити його частиною коду

Завдяки такій роботі з SQL базою мій застосунок не є вразливим до SQL Injection
