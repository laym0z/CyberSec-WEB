API

API - Application Programming Interface слугує як посередник між програмами. В цьому застосунку використовуються API endpoint'и для взаємодії між клієнтом і сервером. До прикладу /api/login - endpoint для авторизації, який обробляє введені користувачем дані та надає або забороняє доступ. Він викликається зі сторінки /login, але також його можна викликати і безпосередньо. наприклад, через термінал командою curl:

curl -k -H "Content-Type: application/json" -d '{"email_or_username": "user", "password": "user"}' https://192.168.100.10:8000/api/login

В моєму випадку це поверне помилку 400, оскільки сервер очікує CSRF токен в header'і. Але за відсутності такої перевірки - команда поверне код 200, сесійний токен та токен запам'ятовування.

Дуже часто у таких endpoint'ів є прив'язка до ролі. Тобто за логікою застосунку він викликається лише якщо авторизований користувач має відповідні права. У цьому застосунку є функція додавання завдань для користувачів, і цю функцію може виконували лише модератор. Але на стороні сервера в цьому API не відбувається перевірка ролі:

@app.route("/api/add_task", methods=["POST"])
@login_required
def add_task():
    data = request.get_json(silent=True)
    ...

Ця функція потребує, аби користувач, який робить звернення, був авторизованим (декоратор @login_required) але знову ж таки, перевірки ролі немає. Саме тому запит до нього може зробити навіть звичайний користувач:

curl -b "session=..." -k -H "Content-Type: application/json" -d '{"task_name":"sdfsf","task_description":"sdfsd","task_assignment":"user"}' https://192.168.100.10:8000/api/add_task

Або API видалення користувача, яка має належати вже адміністратору, але знову не проводить відповідну перевірку:

@app.route("/api/delete_user", methods=["DELETE"])
@login_required
def delete_user():
    data = request.get_json(silent=True)
    if Users.query.filter_by(username=data["username"]).first():
    ...

Для роботи з наведеними вище API достатньо надати cookie звичайного користувача.

Аби уникнути такої участі в моделі таблиці користувачів був створений метод який перевіряє, чи має поточний користувач потрібну роль:

    def has_role(self, role_name: str) -> bool:
        return any(role.name == role_name for role in self.roles)

Далі в потрібних API можна викликати цей метод:

if not current_user.has_role("admin"):
        return jsonify({"status": "error", "message": "this user does not have permission"}), 403

Або замість того аби писати власний метод перевірки можна використати декоратор @roles_required з бібліотеки flask_user.

from flask_user import roles_required

@app.route("/api/delete_user", methods=["DELETE"])
@login_required
@roles_required('admin')
def delete_user():
    data = request.get_json(silent=True)
    if Users.query.filter_by(username=data["username"]).first():
    ...

В разі якщо користувач без належних прав доступу спробує звернутися до API, яке запитує права доступу, то у відповідь отримає помилку 403 Forbidden.
