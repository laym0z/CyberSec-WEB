SSH

SSH є дуже зручним для віддаленої роботи з сервером. Але він також є потенційною точкою входу для небажаних персон. Саме тому перед використанням SSH сервісу на сервері його варто налаштувати задля безпеки.

Всі налаштування будуть відбуватися в конфігураційному файлі:

sudo nano /etc/ssh/sshd_config

Базовими налаштуваннями вважається заборона з'єднання через пароль і тільки через авторизований ключ. Також категорично забороняється з'єднання через SSH як root користувач будь кому.

PasswordAuthentication no  
PermitRootLogin no
MaxAuthTries 3

Також за бажанням можна обмежити кількість сесій та встановити кастомний порт:

maxSessios 2
Port 2222

Задля того, аби тільки потрібні користувачі мали змогу з'єднуватись через SSH треба додати їхні публічні RSA ключи в файл авторизованих ключів на сервері:

~/.ssh/authorized_keys

І що не менш важливо - треба створювати користувачів з обмеженими діями і видавати їм лише необхідні права.
